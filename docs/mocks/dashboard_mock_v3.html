Schema & Integration

i
Fairness reports require metrics, subgroups, and flags. Leakage reports require summary, flagged_features, and notes. The dashboard reads any JSON file in the reports directory matching this contract. No module imports another — JSON is the integration surface.

Fairness report example

{
  "dataset": "Adult Income",
  "sensitive_attribute": "sex",
  "model": "LogisticRegression",
  "metrics": {
    "demographic_parity_ratio": 0.3214,
    "equal_opportunity_difference": -0.1287,
    ...
  },
  "subgroups": [...],
  "flags": [...]
}

Leakage report example

{
  "dataset": "Adult Income",
  "checks_run": [...],
  "summary": {
    "features_checked": 14,
    "flags_raised": 2,
    "risk_level": "Medium"
  },
  "flagged_features": [...],
  "notes": [...]
}
